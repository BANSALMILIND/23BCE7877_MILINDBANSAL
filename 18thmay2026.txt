-- TRUNCATE: The table structure remains, but all data inside is wiped out.
-- It cleans out the table, leaving an empty shell behind.

-- DROP: The entire table structure is completely gone.
-- The complete thing is destroyed from the database permanently.

-- DML (Data Manipulation Language):
-- Used for modifying data, e.g., changing 'Jayanth' -> 'Jayanth V'.
-- DELETE is used for specific rows (deleting one particular row or specific data).

-- PERMANENT ACTIONS (DDL - Cannot be undone!):
-- DROP is a permanent action. If executed, you CANNOT ROLLBACK. 
-- It is gone once, and gone for all life!
-- TRUNCATE is permanent too (cannot be rolled back).

-- UNDO CAPABILITIES (DML):
-- Unlike MS Paint where Undo/Redo always works, SQL has strict rules.
-- Only DELETE and other DML actions can be undone using ROLLBACK.
-- You can undo those specific data actions if autocommit is off.

-- DML(DATA MANIPULATION LANGUAGES).
